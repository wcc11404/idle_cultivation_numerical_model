# 数值建模范围与配置说明

> 世界观与背景设定见 [docs/world_setting.md](./world_setting.md)

## 一期：境界修炼耗时建模

### 建模目的
- 评估玩家从炼气一层到最高层的修炼耗时合理性。
- 对比“灵气获取速度”与“突破材料获取速度”两条时间线的压力差异。
- 支持快速调参并将结果写回本项目 `data/realms.json`。

### 主要假设
- 只建模基础修炼与突破材料，不引入术法增益。
- 材料速率固定：`foundation_herb = 30 / 天`。
- 突破材料按 `realms.json + recipes.json` 递归折算，不使用额外经验规则。

### 可调整配置
- 炼气期逐层属性：`health`、`attack`、`defense`。
- 炼气期逐层：`spirit_energy_cost`、`max_spirit_energy`。
- 炼气期逐层：`spirit_stone_cost`。
- 筑基一层属性基准：`health`、`attack`、`defense`。
- 筑基一层基准：`spirit_energy_cost`、`max_spirit_energy`。
- 筑基一层资源基准：`spirit_energy_cost`、`spirit_stone_cost`、`max_spirit_energy`。
- 递推参数（资源）：层内倍率、跨大境界首层倍率。
- 递推参数（属性）：层内倍率、跨大境界首层倍率。
- 高阶丹方材料参数：低阶丹药数量、破境草数量（支持批量覆盖）。

## 二期：普通历练战斗收益建模

### 建模目的
- 在普通历练区域中评估“不同境界层级角色”的连续作战能力。
- 输出满血开局可持续战斗次数与单位时间收益，支持区域平衡评估。

### 主要假设
- 仅使用普通历练区：`areas.normal_areas`，不包含每日区域与南麓试练塔。
- 玩家属性只用基础属性（境界+层级），不考虑任何术法增益。
- 双方穿透固定为 `0`。
- 攻击技能系数可调，默认 `1.0`。
- 场间等待时间可调，默认 `4` 秒，且不得为负。
- 随机种子固定（用于保证结果可复现），页面不提供调节。
- 仿真采用固定 `3` 轮平均。
- 先求平均总耗时，再额外加 `100` 秒用于折算单场耗时（无损特例除外）。

### 伤害公式
- `伤害 = max(攻击 * (1 - max(防御-穿透,0) / max(防御-穿透+K, K)),1) * 攻击技能系数`

### 可调整配置
- `K`（减伤曲线参数，默认 100）。
- 攻击技能系数（默认 1.0）。
- 场间等待时间（秒，默认 4，最小 0）。

### 固定仿真策略（提速）
- 每次死亡前，最多真实模拟 `30` 场战斗。
- 每个组合最多模拟 `3` 次。
- 若达到场次上限仍未死亡，则按剩余血量比例做线性外推，不继续全量真模拟。
- 若 30 场全程无损，则平均战斗次数展示为 `♾️`，单场耗时按“30场总耗时/30”计算。

### 页面输出
- 自定义组合：任意大境界+层级+普通区域
  - 平均战斗次数
  - 平均总耗时
  - 平均战斗效率（次/小时）
  - 各掉落 item 期望数量/小时
- 批量矩阵：仅炼气/筑基/金丹的 `1/5` 层 × 每个普通区域
  - 列为：大境界层级、区域、平均战斗次数、平均战斗效率、每小时掉落期望数量
  - 掉落期望在单元格内按“中文名: 数量”拼接展示

## 共享口径
- 本项目 `data/*` 为分析真值来源，保存行为只落地模型项目本地数据。
- 页面参数更新后即时重算，保存按钮仅影响对应配置文件。
- 显示格式统一采用“两位小数去尾零”。
